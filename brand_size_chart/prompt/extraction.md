# Table Extraction

Use the configured browser and the discovery evidence to extract the requested size chart into schema-valid rows. Every source page, source asset, and source data load must happen through that browser. Other loading mechanisms are forbidden for source data. Extract only the table whose browser-visible heading, tab, section, or table title matches the requested size_group_key and source title from the stage context. If the source page contains multiple chart tabs, groups, or tables, select the matching target table before reading rows. Do not extract a default, current, product-category, adjacent, or differently named table from the same page. Do not change size_group_key, source_type, source_url, or source_title. Preserve every source table column that belongs to the size chart when that column has a value in the source row. Do not emit measurement entries for blank source cells; describe systematic blank cells in the chart description when needed. For size-system or label-equivalence columns such as regional size, local-market size, alpha size, numeric size, or age labels, keep the column as a measurement and use unit='size' unless the source gives a more specific physical unit. Height, length, weight, body, and garment measurements must keep their physical source unit such as cm, inch, kg, or lb even when the source label is a generic word like size or taille.
